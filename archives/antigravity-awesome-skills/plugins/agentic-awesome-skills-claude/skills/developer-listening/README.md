# Developer Listening

Monitor developer conversations across GitHub, Hacker News, Reddit, Stack Overflow, Twitter, and Discord to understand sentiment, find engagement opportunities, and gather competitive intelligence.

## What This Skill Covers

- Setting up keyword monitoring for brand, competitors, and problem spaces
- Platform-specific monitoring strategies (GitHub, HN, Reddit, SO, Twitter, Discord)
- Sentiment analysis and prioritization frameworks
- Finding high-value engagement opportunities (frustrated users, buy intent, questions)
- Extracting competitive intelligence from developer conversations

## Recommended Tools

- **Social listening tools** — Monitor developer platforms (GitHub, HN, Reddit, SO, Twitter)
- **Platform-specific search** — Native search on each platform for targeted monitoring

## Quick Start

1. Define keyword categories: brand, competitors, problems, buy intent
2. Configure your monitoring tool with keywords
3. Set up daily review of negative sentiment and buy intent mentions
4. Create response playbooks for common engagement scenarios
5. Track trends over time for strategic insights
