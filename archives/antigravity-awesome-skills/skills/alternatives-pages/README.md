# Alternatives Pages

Create "[Competitor] alternative" and comparison pages for developer tools that rank for competitive keywords and convert developers honestly.

## What This Skill Covers

- SEO research for alternative and comparison keywords
- Page structure that converts without being sleazy
- Building honest, verifiable comparison tables
- Migration content and addressing switching concerns
- When to name competitors vs stay general
- Legal considerations for competitive content
- Ongoing content maintenance

## Recommended Tools

- **Social listening tools** — Monitor competitor conversations and developer pain points
- **SEO tools** (Ahrefs, Semrush) — Keyword research and search volume

## Quick Start

1. Research competitor pain points and comparison patterns using social listening
2. Prioritize pages based on search volume and competitive overlap
3. Create honest, detailed comparison content
4. Include migration guidance and support
5. Set up monitoring for content freshness triggers
6. Establish quarterly review cadence

## Page Types

**Alternatives pages:** "[Competitor] Alternative" - Target developers looking to switch
**Comparison pages:** "[You] vs [Them]" - Target developers actively evaluating
**Category pages:** "Best [Category] Tools" - Target category researchers

## Key Principles

- Be honest - developers fact-check everything
- Be helpful - even if they don't choose you
- Be specific - vague comparisons waste time
- Be current - date comparisons and keep them fresh
