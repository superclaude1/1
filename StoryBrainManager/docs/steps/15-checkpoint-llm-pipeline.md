# 步骤 15【检查点】：文本转结构化脚本管线验收

## 目标
串联步骤 11~14，完成「配置Key → 输入小说文本 → 提取角色对话 → 界面校对展示」的完整
验收，包括正常路径和异常路径两条链路。

## 涉及文件
无新增文件，纯集成验证。

## 验证清单（端到端操作）
1. Settings 页配置真实 DeepSeek API Key（复用步骤 11）
2. 用步骤 12 的"测试连接"按钮确认网络链路通畅
3. 进入 ProjectWorkspace，粘贴一段 500 字左右的小说片段
4. 点击"提取"，等待 loading 结束（复用步骤 14）
5. 核对界面展示的对话行：角色归属、文本内容、语气标签是否合理（复用步骤 13）
6. 修改 Settings 中的 Key 为一个错误值（或断网），重新点击提取，确认：
   - 界面显示明确错误提示，而非白屏或卡死
   - 应用整体仍可正常操作其他功能（如切换页面、录音等不受影响）
7. 恢复正确 Key，再次提取，确认功能恢复正常

## 完成标准 (DoD)
- [ ] 正常路径：配置Key → 提取 → 展示结果，全程顺畅无报错
- [ ] 异常路径：错误Key/断网时有清晰反馈，且不影响应用其他功能继续使用
- [ ] 两条路径切换（先异常后正常，或反之）均表现正确，无需重启应用
