# 步骤 09：多段合并链（3段以上）验证

## 目标
验证 `merge_many` 对 3 段及以上音频的连续合并结果正确，且合并过程中产生的中间临时
文件（`.tmp` / `.tmp2`）被正确清理，不遗留垃圾文件。

## 涉及文件
无新增文件，主要针对已有的 `src-tauri/src/audio/merge.rs` 里的 `merge_many` 函数
做集成测试。

## 实现要点
`merge_many` 已实现为链式合并（A+B → tmp，tmp+C → tmp2，重命名为最终输出），本步骤
重点验证：
- 链式合并顺序是否严格保持 `track_paths` 数组的原始顺序
- 每一轮产生的中间文件是否被 `std::fs::remove_file` 正确删除

## 验证方法
1. 用统一质量档位（如 High）连续录制 4 段内容不同的音频：A="一"，B="二"，C="三"，
   D="四"
2. 一次性调用合并（`mergeTracks([pathA, pathB, pathC, pathD], outputPath)`）
3. 播放最终生成的文件，应完整依次听到"一、二、三、四"，顺序不能错乱
4. 打开输出目录，用文件资源管理器确认：
   - 只存在最终的 `outputPath` 对应文件
   - 没有残留 `*.tmp` / `*.tmp2` 等中间文件
5. 重复执行 2~3 次（多次合并同一批文件到不同输出路径），确认结果稳定一致，没有
   偶发的顺序错乱或残留文件积累

## 完成标准 (DoD)
- [ ] 4 段音频合并后播放顺序完全正确（一→二→三→四）
- [ ] 输出目录中无残留临时文件
- [ ] 多次重复执行结果稳定，无随机性错误
- [ ] 合并总耗时随文件数量增长基本呈线性，无明显性能异常
