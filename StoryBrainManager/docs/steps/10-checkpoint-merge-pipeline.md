# 步骤 10【检查点】：音频合并管线验收

## 目标
串联步骤 06~09，完整走一遍「多段录制 → 格式校验 → 无损合并 → 完整播放」的音频处理
管线，作为 Phase 1 音频子系统的阶段性验收。

## 涉及文件
无新增文件，纯集成验证，可结合步骤 05 已经打通的 ProjectWorkspace 录制/播放界面。

## 验证清单（端到端操作）
1. 用统一质量档位录制 3 段内容不同的音频（复用步骤 04/06）
2. 尝试混入 1 段用不同质量档位录制的音频，一起发起合并，确认被正确拦截并给出清晰
   错误提示（复用步骤 08）
3. 移除格式不一致的那一段，仅用 3 段格式一致的音频重新发起合并（复用步骤 07/09）
4. 播放合并结果：
   - 内容顺序正确，与录制顺序一致
   - 总时长约等于 3 段时长之和
   - 用十六进制工具核对文件头字节，`riff_size`/`data_size` 与公式吻合（复用步骤 07
     的校验方法）
5. 确认合并过程中无残留临时文件（复用步骤 09）

## 完成标准 (DoD)
- [ ] 全流程仅通过 UI 操作 + 文件工具校验即可完成，无需查看代码
- [ ] 格式校验、正常合并、多段合并三条路径均验证通过
- [ ] 无残留临时文件、无控制台报错
- [ ] 该阶段验收通过后，音频处理能力视为对文档中"纯 Rust 无损合并规避 SELinux
      限制"方案的 Windows 端等价实现已完成，可直接复用到 Phase 2 Android 端
