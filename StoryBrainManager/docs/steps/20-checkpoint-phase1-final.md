# 步骤 20【检查点】：Phase 1 全流程验收（最终交付标准）

## 目标
不看代码、不看实现细节，完整走一遍产品级用户操作流程，作为 VoxNovel Windows 桌面版
Phase 1 的最终验收标准。通过即代表可以进入 Phase 2（Android 移植）。

## 涉及文件
无新增文件，全流程集成验收。

## 验证清单（从零开始，模拟真实用户操作）
1. 双击打开 VoxNovel（`npm run tauri build` 打出的安装包，或 `npm run tauri dev`），
   窗口正常弹出（复用步骤 01）
2. Home 页点击"新建工程"，输入工程名，粘贴一段小说文本（复用步骤 17）
3. 若尚未配置，进入 Settings 配置真实 DeepSeek API Key（复用步骤 11）
4. 回到 ProjectWorkspace，点击"提取"，看到结构化对话列表（复用步骤 13/14）
5. 逐行录制角色朗读，可尝试用不同语速/语气表现（复用步骤 04/18）
6. 点击"合成完整音频"，生成最终广播剧片段（复用步骤 19）
7. 播放最终音频，确认内容、顺序、衔接自然，无杂音或断层（复用步骤 07/09/19）
8. 完全关闭应用后重新打开，确认该工程、录制的分轨、最终合成音频均依然存在，可以
   继续编辑或直接播放（复用步骤 16）
9. 全程操作中，控制台/终端无未捕获异常，无白屏，无需重启应用即可恢复的错误提示
   均有合理反馈（复用步骤 08/15 验证过的异常路径）
